JNDIMock4S

JNDI mocking for scala integration tests - especially that requires mocking of resources such as connection pool datasource, JMS Queues, Topics etc. For more on JNDI refer to: http://docs.oracle.com/javase/jndi/tutorial/

Quick Start

To use JNDIMock4S - simply add following to your build.sbt file - 

libraryDependencies ++= Seq( "com.github.anirudhvyas" % "jndimock4s_2.11" % "1.0")

Next lets build a simple test case with binding of a data source (I have tried to keep it as simple as possible withtout any further complexity). 

class SomeSampleTest {
  var _ds: javax.sql.DataSource = null  // a JDBC Data source that application persistence tier uses 
  def ds = _ds
  def ds_=(value: DataSource) = _ds = value

  var _jndiMockFactory: JNDIMockFactory = null  // factory that will bind it.
  def jndiMockFactory = _jndiMockFactory
  def jndiMockFactory_=(value: JNDIMockFactory) = _jndiMockFactory = value

  def setUp(): Unit = {
     println("test setup beginning  ...")
    val dataSource = new MysqlConnectionPoolDataSource()
    dataSource.setEmptyStringsConvertToZero(false)
    dataSource.setParanoid(true)
    dataSource.setUser("root")
    dataSource.setDatabaseName("calculator")
    ds = dataSource
    jndiMockFactory = new JNDIMockFactory
    jndiMockFactory.ctx.bind("jdbc/calculatorDS", ds)
    System.setProperty(javax.naming.Context.INITIAL_CONTEXT_FACTORY, classOf[JNDIMockFactory].getName)
    println("test setup completed!  ...")
  }
}

Past setup you are free to choose framework of your choice below to test stuff that requires database access.
