scala-numerals

Handy way of doing numbers like you can do times with duration._

Simply import numerals._ and then a bunch of implicits are available, eg:

44.hundred
20.thousand
100.million
99.billion

You can apply the usual mathematical operators to these:

44.hundred + 10.thousand
20.thousand - 61.hundred
100.million / 2.thousand
99.billion * 10
-41.thousand // unary negation

Ordering is supported, viz:

Seq(1.million, 9.thousand, 5.units, 10.hundreds).sorted

As is the Numeric typeclass, viz:

Seq(1.million, 9.thousand, 5.units, 10.hundreds).sum
Seq(4.hundreds, 14.millions, 44.hundreds).max
Seq(2.billion, 1151.millions,  154141.thousands).min

Equals is overridden to allow for comparison between primitives and numerals. If using primitives, then the comparision is done in base units. Eg:

Seq(1.thousand, 2.hundred, 3.units).sum == 1203.units
Seq(1.thousand, 2.hundred, 3.units).max == 1000
Seq(1.thousand, 2.hundred, 3.units).min == 3

Using scala-numerals in your project

For SBT users simply add:

libraryDependencies += "com.sksamuel.scala.commons" %% "scala-numerals" % "0.90"

For Maven users simply add (replace 2.11 with 2.10 for Scala 2.10):

<dependency>
    <groupId>com.sksamuel.scala.commons</groupId>
    <artifactId>scala-numerals_2.11</artifactId>
    <version>0.90</version>
</dependency>

And finally gradlers, should do:

compile("com.sksamuel.scala.commons:scala-numerals_2.11:0.90")
