SI-2712-fix plugin

This is proof of concept fix to SI-2712 implemented as a compiler plugin. There is also a PR to fix this directly in scalac.

The implementation is based on a simple algorithm as suggested by Paul Chiusano in the comments on SI-2712: Treat the type constructor as curried and partially applied, we treat a prefix as constants and solve for the suffix. For the example in the ticket, unifying M[A] with Int => Int, this unifies as,

M[t] = [t][Int => t]
A = Int

More detailed explanations can also be found at this gist Explaining Miles's Magic by @djspiewak and the readme of the demo project.

One place to discuss this fix is at typelevel's gitter room.

Usage

Update ...

This plugin is deprecated. Please use Typelevel Scala with the -Ypartial-unification flag instead.

Binary release artefacts are published to the [Sonatype OSS Repository Hosting service][sonatype] and synced to Maven Central. To use the plugin in your project add the following to its sbt build file, ```scala addCompilerPlugin("com.milessabin" % "si2712fix-plugin" % "1.2.0" cross CrossVersion.full) ``` If you intended to use the `@unifyRightToLeft` annotation to enable right-to-left unify rule please add the following as well ```scala libraryDependencies += "com.milessabin" % "si2712fix-library" % "1.2.0" cross CrossVersion.full ``` More context about right to left rule can be found in this [issue comment][right-left].

Building the plugin

This plugin is built with SBT 0.13.11 or later, and its master branch is built with Scala 2.11.8 and 2.10.6 by default.
